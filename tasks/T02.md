## O que entregar

### 1. Password (Argon2id + Pepper)
- `internal/auth/password.go` — HashPassword(password, pepper), VerifyPassword(password, hash, pepper)
- Pepper vem do config (env var PASSWORD_PEPPER)
- Params: time=1, memory=64MB, threads=4, keyLen=32

### 2. JWT (Ed25519)
- `internal/auth/jwt.go` — GenerateAccessToken(userID, orgID, role) → string, ValidateAccessToken(token) → Claims
- Claims: Sub, Org, Role, Exp, Iat, Jti. Expiração: 15min.

### 3. Auth Middleware
- `internal/middleware/auth.go` — extrai Bearer token, valida, coloca claims no context. 401 se inválido.

### 4. RBAC Middleware
- `internal/middleware/rbac.go` — RequireRole(roles ...string). 403 se role não permitido.

### 5. Auth Endpoints
- `internal/auth/handler.go` — HandleLogin, HandleRefresh, HandleLogout
- `internal/auth/service.go` — Login, RefreshToken, Logout
- `internal/auth/repository.go` — FindUserByEmail, CRUD refresh_tokens

**Refresh Token Rotation:** cada refresh gera novo par. Token revogado reusado → revoga família inteira.
Refresh token armazenado como hash SHA-256 no banco.

### 6. Audit Log Service
- `internal/audit/service.go` — Log(ctx, Entry) faz INSERT. Nunca bloqueia operação principal se falhar.
- `internal/audit/repository.go` — Insert (only), List (com filtros + pagination)

Registrar audit em: login, logout, refresh.
