## O que entregar

### 1. CRUD básico
- `internal/asset/` — handler, service, repository, dto
- Create: status=draft, version=1, QR code do client
- Update: só draft/rejected. TECH só próprios, ADMIN qualquer.
- Delete: só draft. Soft delete. TECH só próprios.
- List: filtros (status, type_id, created_by) + paginação
- Get: com TypeRef + UserRef populados

### 2. PostGIS
- Insert: `ST_MakePoint(longitude, latitude)::geography` — ATENÇÃO: longitude primeiro!
- Select: `ST_Y(location::geometry) AS latitude, ST_X(location::geometry) AS longitude`

### 3. Fluxo de aprovação
- POST /{id}/submit — draft→pending (TECH dono ou ADMIN)
- POST /{id}/approve — pending→approved (ADMIN only, preenche approved_by)
- POST /{id}/reject — pending→rejected (ADMIN only, reason obrigatório)
- Transição inválida → 409 INVALID_STATUS_TRANSITION

### 4. Versionamento
Editar asset approved → cria novo registro com parent_id + version+1, status=draft.
GET /{id}/history — cadeia de versões (parent chain)

### 5. Busca geográfica
GET /assets/nearby?lat=X&lng=Y&radius_m=Z — ST_DWithin, apenas approved, com distance_m
