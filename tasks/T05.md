# T05 — Media (Upload)

> **Ler:** RULES.md + API_CONTRACTS.md (seção 10: Media) + DATABASE.md (media)
> **Depende de:** T04

## O que entregar

- `internal/media/handler.go` — HandleUploadURL, HandleConfirm, HandleGet, HandleDelete
- `internal/media/service.go` — GenerateUploadURL, Confirm, GetByID, SoftDelete
- `internal/media/repository.go` — Insert, FindByID, Update, SoftDelete, ListByAsset
- `internal/media/s3.go` — GeneratePresignedPutURL, HeadObject (verifica se arquivo existe)

### Fluxo
1. POST /media/upload-url → valida (mime_type, size_bytes, idempotency_key), cria registro com upload_status=pending, gera presigned PUT URL (15min)
2. Client faz PUT direto no S3 com o arquivo
3. POST /media/{id}/confirm → backend verifica que objeto existe no S3 via HeadObject, atualiza upload_status=uploaded
4. GET /media/{id} → retorna presigned GET URL (1h expiry) gerada sob demanda

### Validação
- mime_type: image/jpeg, image/png, image/webp
- size_bytes: 1 a 10MB (10485760)
- Max 20 fotos por asset

Storage key format: `{org_id}/assets/{asset_id}/{media_id}.{ext}`
