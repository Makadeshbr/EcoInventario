# T06 — Manejos + Monitoramentos

> **Ler:** RULES.md + API_CONTRACTS.md (seções 7–8) + DATABASE.md (manejos, monitoramentos)
> **Depende de:** T04, T05

## O que entregar

### 1. Manejos
- `internal/manejo/` — handler, service, repository, dto
- CRUD: create, list (por asset + org), get, update, soft delete
- Fluxo aprovação: submit, approve, reject (mesmo padrão do asset)
- before_media_id e after_media_id referenciam media do mesmo asset
- GET /assets/{asset_id}/manejos — lista manejos do asset

### 2. Monitoramentos
- `internal/monitoramento/` — handler, service, repository, dto
- CRUD: create, list (por asset + org), get, update, soft delete
- Fluxo aprovação: submit, approve, reject
- health_status: healthy, warning, critical, dead (validar no DTO)
- GET /assets/{asset_id}/monitoramentos — lista do asset

### Regras compartilhadas (mesmas do asset)
- TECH cria e edita próprios (draft/rejected). ADMIN tudo.
- Delete só draft. Soft delete.
- Registros approved são imutáveis (sem versionamento para manejo/monitoramento)
- Audit log em tudo.
