# T07 — Sync Engine (Push + Pull)

> **Depende de:** T04, T06

## O que entregar

### 1. Sync Push
- `internal/sync/handler.go` — HandlePush
- `internal/sync/service.go` — ProcessPush
- `internal/sync/idempotency.go` — CheckAndStore, Cleanup

POST /sync/push recebe batch de operations (max 50).
Para cada operação:
1. Verifica idempotency_key → se já processada, retorna resultado anterior (status=duplicate)
2. Se CREATE: chama service.Create da entidade correspondente
3. Se UPDATE: compara client_updated_at com server updated_at → diverge = conflict
4. Se DELETE: chama service.SoftDelete
5. Armazena idempotency_key + resultado

Response: array de results com status (ok, conflict, error, duplicate) por operação.

### 2. Sync Pull
- `internal/sync/handler.go` — HandlePull

GET /sync/pull?since={ISO8601}&limit=100&cursor=...
Retorna registros (assets, manejos, monitoramentos) com updated_at > since.
Inclui soft-deleted (action=delete) para que o mobile saiba remover.

### 3. Cleanup job
Goroutine que roda periodicamente (24h) e remove idempotency_keys > 30 dias.
