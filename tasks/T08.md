## O que entregar

### 1. Public API (sem autenticação)
- `internal/public/handler.go` — HandleListAssets, HandleGetAsset, HandleListAssetTypes, HandleResolveQR
- `internal/public/service.go` — busca apenas dados com status=approved
- `internal/public/repository.go` — queries otimizadas para público

Endpoints:
- GET /api/v1/public/asset-types — lista tipos (para filtro)
- GET /api/v1/public/assets?bounds=sw_lat,sw_lng,ne_lat,ne_lng — assets no bounding box
- GET /api/v1/public/assets/{id} — ficha completa: asset + media + manejos + monitoramentos (tudo approved)
- GET /api/v1/public/assets/resolve-qr?code=... — resolve QR → asset_id

**Excluir dos responses:** created_by, approved_by, rejection_reason, status, notes internas, gps_accuracy, version, parent_id.
**Incluir:** organization_name, fotos (presigned URLs), manejos (descrição + fotos), monitoramentos (notes + health_status).

### 2. Health Check
- GET /api/v1/health — verifica DB + S3. Retorna 200/503.

### 3. Rate Limiting
- `internal/middleware/rate_limit.go` — sliding window in-memory
- Login: 5/min por IP. Sync: 30/min por user. Upload: 60/min por user. Demais: 120/min por user. Public: 60/min por IP.
- 429 + Retry-After header quando excedido.
