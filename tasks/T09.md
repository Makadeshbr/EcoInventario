> **Depende de:** T02 (backend rodando)

## O que entregar

### 1. Setup Expo
- Criar projeto Expo SDK 52 com TypeScript
- Instalar deps: expo-router, expo-sqlite, zustand, @tanstack/react-query, ky, react-native-mmkv, zod, uuid, react-native-get-random-values, @react-native-community/netinfo, expo-camera, expo-location, expo-image-manipulator
- `tsconfig.json` com path aliases (@/)
- `app.json` configurado

### 2. SQLite setup
- `src/db/database.ts` — abrir conexão expo-sqlite
- `src/db/migrations.ts` — criar tabelas locais: assets, media, manejos, monitoramentos, sync_queue, media_upload_queue, sync_metadata, sync_conflicts (conforme DATABASE.md seção 3)
- Rodar migrations no startup do app

### 3. Tipos TypeScript
- `src/types/domain.ts` — Asset, AssetType, Media, Manejo, Monitoramento, User + enums (conforme MOBILE_GUIDE.md)
- `src/types/api.ts` — ApiError, PaginatedResponse, LoginRequest/Response, UploadUrlRequest/Response
- `src/types/sync.ts` — SyncQueueItem, SyncPushResponse, SyncPullResponse, SyncStatus
- `src/constants/config.ts` — todas as constantes (URLs, limites, intervalos)

### 4. API Client
- `src/api/client.ts` — instância ky com prefixUrl, timeout, interceptor de auth (Bearer token)
- `src/api/transforms.ts` — toCamelCase(), toSnakeCase()

### 5. Auth Store
- `src/stores/auth-store.ts` — Zustand + MMKV: accessToken, refreshToken, user, setAuth, refreshAccessToken, logout

### 6. Sync Store
- `src/stores/sync-store.ts` — Zustand: status, lastSyncAt, counts

### 7. Estrutura de navegação (Expo Router)
Criar TODOS os arquivos de layout e telas como placeholder (componente vazio com nome da tela):
- `app/_layout.tsx` — root
- `app/(welcome)/` — boas-vindas
- `app/(guest)/` — tabs visitante (3 tabs: map, scanner, about)
- `app/(auth)/` — login
- `app/(app)/` — tabs profissional (4 tabs: home, assets, scanner, profile) para usuário `tech`

### 8. Utilitários
- `src/utils/uuid.ts` — generateUUID()
- `src/utils/image-compression.ts` — compressImage()
- `src/hooks/use-network-status.ts`
