# T10 — Mobile: Splash + Welcome + Login

> **Ler:** RULES.md + APP_FLOW.md (seções 2.1–2.2) + MOBILE_GUIDE.md (auth store)
> **Depende de:** T09

## O que entregar

### 1. Splash Screen
- Logo + loading indicator
- Verifica se há token salvo no MMKV
- Token válido → navega para (app) (profissional)
- Sem token → navega para (welcome)
- Tenta validar token fazendo refresh silencioso

### 2. Welcome Screen (Boas-vindas)
- `app/(welcome)/index.tsx`
- Branding do app + descrição breve
- Dois botões: **"Explorar"** → (guest), **"Entrar"** → (auth)/login
- Design conforme Stitch (pixel-perfect)

### 3. Login Screen
- `app/(auth)/login.tsx`
- Form: email + senha
- Validação com Zod (email format, senha min 8)
- Chama POST /auth/login
- Sucesso com `role=tech` → salva tokens no auth store → navega para (app)
- Sucesso com `role!=tech` → não entra no modo profissional e mostra mensagem para usar o dashboard web
- Erro → mostra mensagem
- Link "Voltar" → welcome
- Loading state no botão durante request
