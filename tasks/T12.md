> **Depende de:** T10, T09

## O que entregar

### 1. Home
- `app/(app)/(home)/index.tsx`
- Resumo: assets recentes (últimos 5 locais), indicador de sync, contadores (draft, pending, approved, rejected)
- Cards clicáveis → navegam para listas
- Dados do SQLite local (funciona offline)

### 2. Lista de Assets
- `app/(app)/(assets)/index.tsx`
- Lista do SQLite local com filtros (status, tipo)
- Pull-to-refresh (dispara sync se online)
- FAB (floating action button) → criar novo
- Badge "não sincronizado" em itens com is_synced=false
- Empty state: ilustração + "Nenhum asset cadastrado" + CTA criar

### 3. Criar Asset (multi-step)
- `app/(app)/(assets)/new.tsx`
- Step 1: selecionar tipo (de asset_types locais) + notas
- Step 2: capturar localização GPS (expo-location) + exibir mapa com marker + accuracy
- Step 3: tirar fotos (expo-camera) → comprimir (max 1920px, JPEG 80%)
- Step 4: revisão (todos os dados) + botão "Salvar"
- Salvar: insere no SQLite (status=draft) + gera QR code (UUID) + enfileira no sync_queue + enfileira fotos no media_upload_queue
- Funciona 100% offline

### 4. Detalhes do Asset
- `app/(app)/(assets)/[id]/index.tsx`
- Dados completos do SQLite + galeria de fotos
- Mapa com marker
- Status badge (colorido por status)
- Se draft: botão "Enviar para aprovação" (muda status local para pending + enfileira sync)
- Se rejected: mostra motivo + botão "Editar"
- Se approved: dados read-only

### 5. Editar Asset
- `app/(app)/(assets)/[id]/edit.tsx`
- Mesmo form de criação, preenchido com dados existentes
- Só acessível se status=draft/rejected E user é dono
- Salva → atualiza SQLite + enfileira UPDATE no sync_queue
