## O que entregar

### 1. Sync Engine Core
- `src/sync/sync-engine.ts` — classe com start(), stop(), sync()
- Roda a cada 30s quando online. Ordem: push metadata → push media → pull.
- Atualiza sync store com progresso

### 2. Push Metadata
- Lê sync_queue (status=pending), monta batch (max 50), chama POST /sync/push
- Processa resultados: ok→synced, duplicate→synced, conflict→conflict, error→retry
- Retry com backoff exponencial (1s base, 5min max, jitter ±25%)
- Max 5 retries metadata

### 3. Push Media
- Lê media_upload_queue (status=pending), processa 1 por vez
- Para cada: POST /media/upload-url → PUT no S3 → POST /media/{id}/confirm
- Max 3 retries mídia

### 4. Pull Changes
- GET /sync/pull?since=last_sync_at — pagina até has_more=false
- Aplica changes no SQLite (upsert ou soft delete)

### 5. Conflict Handler + UI
- `src/sync/conflict-handler.ts`
- `app/(app)/(profile)/conflicts/index.tsx` — lista de conflitos
- `app/(app)/(profile)/conflicts/[id].tsx` — side-by-side (local vs servidor)
- Ações: aceitar servidor, reenviar local, merge manual

### 6. Scanner QR (profissional)
- `app/(app)/(scanner)/index.tsx`
- Lê QR → busca no SQLite local primeiro → se não achar, busca na API
- Navega para detalhes do asset (modo profissional, com ações)

### 7. Perfil + Sync Status
- `app/(app)/(profile)/index.tsx` — nome, email, role, org + botões (sync, config, logout)
- `app/(app)/(profile)/sync.tsx` — contadores de fila, último sync, botão "Sincronizar agora"
- Fluxo profissional restrito a usuário `tech`

### 8. Configurações + Sobre
- `app/(app)/(profile)/settings.tsx` — qualidade foto, intervalo sync, limpar cache
- `app/(app)/(profile)/about.tsx` — versão, org, suporte
