# T15 — Dashboard: Setup + Layout + Auth + Home

> **Ler:** RULES.md + APP_FLOW.md (seção 3: Dashboard)
> **Depende de:** T02 (backend rodando)

## O que entregar

### 1. Setup Next.js
- Criar projeto Next.js 15 (App Router) + TypeScript + Tailwind CSS 4
- Instalar deps: @tanstack/react-query, zod, leaflet, react-leaflet, recharts
- Tipos TypeScript compartilhados (copiar do mobile ou criar shared)
- API client (fetch wrapper com JWT em cookie httpOnly ou header)

### 2. Auth
- `/login` page — email + senha → POST /auth/login
- JWT armazenado em cookie httpOnly (ou memória + refresh)
- Middleware Next.js para proteger rotas /dashboard/*
- Redirect para /login se não autenticado

### 3. Layout
- `/dashboard/layout.tsx`
- Sidebar fixa com links de navegação
- Role-based: esconde links que o role não pode acessar (ex: Usuários só para ADMIN)
- Header com nome do user + botão logout
- Responsivo (sidebar colapsável no mobile)

### 4. Home
- `/dashboard/page.tsx`
- Métricas: total assets, pendentes de aprovação, aprovados, rejeitados
- Gráficos (Recharts): assets por status (pie/donut), assets por tipo (bar), atividade mensal (line)
- Cards clicáveis: "X pendentes de aprovação" → link para /dashboard/approval
- Dados via API: GET /api/v1/stats
