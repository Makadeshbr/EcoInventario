# T16 — Dashboard: Aprovação + Assets + Mapa

> **Ler:** RULES.md + APP_FLOW.md (seção 3.2: telas 3–5, 8) + API_CONTRACTS.md
> **Depende de:** T15

## O que entregar

### 1. Fila de Aprovação (ADMIN only)
- `/dashboard/approval/page.tsx`
- Tabela de registros PENDING (assets + manejos + monitoramentos)
- Filtros: tipo de entidade, data, técnico responsável
- Expandir row: ver fotos, localização no mapa, dados completos
- Ações inline: botão Aprovar (chama POST /{id}/approve) + botão Rejeitar (abre modal com textarea para motivo → POST /{id}/reject)
- Após ação: remove da lista com feedback visual

### 2. Assets Lista
- `/dashboard/assets/page.tsx`
- Tabela paginada (cursor-based) com colunas: tipo, QR code, status (badge), técnico, data
- Filtros: status, tipo, técnico, busca por QR code
- Toggle vista: lista ↔ mapa (reutiliza componente de mapa)
- Click na row → navega para detalhes

### 3. Asset Detalhes
- `/dashboard/assets/[id]/page.tsx`
- Dados completos do asset
- Galeria de fotos (grid + lightbox ao clicar)
- Mapa com marker da localização
- Histórico de versões (GET /{id}/history) como timeline
- Lista de manejos vinculados (com fotos before/after)
- Lista de monitoramentos vinculados (com health_status colorido)
- Audit log do asset (GET /audit-logs?entity_type=asset&entity_id=X)

### 4. Mapa Interativo
- `/dashboard/map/page.tsx`
- Leaflet com todos os assets da organização
- Clusters para alta densidade
- Filtros: status, tipo, técnico
- Click no marker → popup com resumo → link "Ver detalhes"
