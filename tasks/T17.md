## O que entregar

### 1. Manejos Lista
- `/dashboard/manejos/page.tsx`
- Tabela paginada: asset (link), descrição (truncada), status (badge), técnico, data
- Filtros: status, asset, técnico, data
- Expandir row ou modal: fotos before/after **side-by-side**
- Se ADMIN + status pending: ações aprovar/rejeitar inline (mesma UX da fila de aprovação)

### 2. Monitoramentos Lista
- `/dashboard/monitoramentos/page.tsx`
- Tabela paginada: asset (link), health_status (badge colorido: verde/amarelo/vermelho/preto), notas (truncadas), técnico, data
- Filtros: status, health_status, asset, técnico, data
- Health status visual: dot ou badge com cor + label
- Se ADMIN + status pending: ações aprovar/rejeitar inline

### 3. Componentes compartilhados
- `StatusBadge` — badge colorido por status (draft=gray, pending=yellow, approved=green, rejected=red)
- `HealthBadge` — badge colorido por health (healthy=green, warning=yellow, critical=red, dead=black)
- `ApprovalActions` — botões aprovar/rejeitar + modal de motivo (reutilizável)
- `PhotoComparison` — side-by-side de duas fotos (before/after)
